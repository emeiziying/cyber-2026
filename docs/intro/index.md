# 开始阅读

`Cyber 2026` 是一套面向团队内部的赛博飞升方法论。它不试图承载真实业务功能，而是把团队实现“赛博飞升”所需的方法讲清楚：

- 为什么值得把更多工作交给 AI
- 什么任务该停在 prompt / rule / MCP / skill / hook
- 什么任务值得继续升级成 agent
- 如何把这些能力安全接入团队协作流程

## 这个站点怎么读

### 学习主线

1. 从 [范式转变](/paradigm-shift/) 开始，先建立认知框架
2. 继续读 [Rules](/rules/) 到 [Agent 开发](/agent-development/)，判断规则、工具、工作流与 agent 的边界
3. 最后读 [生产落地与治理](/production-governance/) 和 [团队工作流与质量控制](/team-workflow/)，把个人方法升级成团队机制

### 辅助入口

- [附录：案例与延伸阅读](/appendix-case-studies/)：跨角色阅读材料、案例演练和复盘模板
- [分享与培训](/presentation-outlines/)：培训大纲、默认 PPT 成品和生成脚本
- [Minimal Agent Demo](/examples/minimal-agent-demo/)：最小示例项目与相关下载资源

## 本站不做什么

- 不承载业务专项 skill / agent 本体实现
- 不提供生产级 MCP 服务、项目私有配置、密钥和环境
- 不充当真实团队项目中的运行时资产仓

如果你想看这套材料的内容分层和辅助资料入口，继续阅读 [内容地图](/project-overview/)。
