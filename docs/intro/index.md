# 开始阅读

`Cyber 2026` 是一套面向团队内部的 AI 落地方法文档。它关注的不是“怎么多用一个工具”，而是怎么把 AI 从零散使用，推进到可复用、可协作、可治理的团队能力。

## 这站解决什么问题

很多团队已经开始用 AI，但常见状态仍然是：

- 会单点提问，不会稳定交付
- 会让 AI 生成内容，不会给边界和验收
- 会各自摸索，不会把经验沉淀成共享规则和流程

这套文档要解决的是四件事：

- 为什么值得把更多工作交给 AI
- 什么任务该停在 prompt / rule / MCP / skill，什么任务值得升级成 agent
- 如何把个人经验沉淀成可复用的数字资产
- 如何把这些能力接进团队协作，而不是停留在个人技巧

## 什么是赛博飞升

“赛博飞升”有两层含义。

**第一层：团队 AI 能力升级。**
重点不是“谁更会写 prompt”，而是让团队逐步具备三层能力：

- 认知转变、协作方式、工具判断
- rule、MCP、skill、agent 的分层构建
- 治理与工作流，把个人做法变成团队机制

**第二层：把经验沉淀成数字资产。**
稳定的判断、约束和流程，不应只留在个人脑子里，而要沉淀成可复用资产：

- 反复出现的约束写成 rule
- 稳定流程封装成 skill 或自动化
- 高价值能力升级成 agent

目标不是替代人或裁员，而是把人从重复执行里抽出来，更多投入判断、决策和创造。

## 推荐怎么读

如果你第一次来，按这个顺序读：

1. 先读 [范式转变](/paradigm-shift/)、[Vibe Coding](/vibe-coding/) 和 [工具全景](/tools-overview/)，建立判断框架
2. 再读 [Rules](/rules/)、[MCP](/mcp/)、[Skills & Hooks](/skills-hooks/)、[Agent 开发](/agent-development/) 和 [Harness Engineering](/harness-engineering/)，理解能力构建路径
3. 最后读 [生产落地与治理](/production-governance/) 和 [团队工作流与质量控制](/team-workflow/)，把个人做法升级成团队机制

## 按目标选择入口

- 想先建立判断框架：从 [范式转变](/paradigm-shift/) 开始
- 想尽快进入实践：从 [Rules](/rules/) 开始
- 想看真实任务怎样落地：看 [附录：案例与延伸阅读](/appendix-case-studies/) 和 [Minimal Agent Demo](/examples/minimal-agent-demo/)
- 想做培训或拿现成材料：看 [分享与培训](/presentation-outlines/) 和 [下载资源](/downloads/)
- 不知道从哪开始：看 [快速启动指南](/intro/quick-start)

## 本站不做什么

- 不提供业务专项 skill / agent 的生产实现
- 不提供项目私有配置、密钥或生产级 MCP 服务
- 不代替真实项目仓的运行时资产管理
