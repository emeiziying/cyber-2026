# 开始阅读

`Cyber 2026` 是一套面向团队内部的赛博飞升方法论。它不试图承载真实业务功能，而是把团队实现”赛博飞升”所需的方法讲清楚。

## 什么是赛博飞升

赛博飞升包含两层含义：

**第一层：团队 AI 能力的逐级跃迁。** 从个人随意使用 AI 工具，逐步升级到有规则、有治理、可量化的团队级 AI 协作体系。对应本站的学习主线：

- 认知转变 → 协作方式 → 工具选型（筑基：个人能做对）
- Rules → MCP → Skills & Hooks → Agent（修炼：能力逐层解锁）
- 生产治理 → 团队工作流（飞升：组织级稳定运转）

**第二层：把人的能力沉淀为可复用的数字资产。** 通过将个人经验抽象成 Rules、Skill 和 Agent，团队的核心能力不再只存在于某个人的脑子里。即使关键成员离开，这些”赛博员工”仍然可以复刻其工作方式，持续为团队产出价值。

简单说：不是”用上 AI”就算飞升，而是把团队知识从人脑搬到可执行的数字形态，让能力属于组织而不只属于个人。

## 这个站点讲什么

- 为什么值得把更多工作交给 AI
- 什么任务该停在 prompt / rule / MCP / skill / hook
- 什么任务值得继续升级成 agent
- 如何把这些能力安全接入团队协作流程

## 这个站点怎么读

建议从 [范式转变](/paradigm-shift/) 开始，沿主线阅读到 [团队工作流与质量控制](/team-workflow/)。完整的内容分层、辅助资料入口和按目标选路径的建议，见 [内容地图](/project-overview/)。

## 本站不做什么

- 不承载业务专项 skill / agent 本体实现
- 不提供生产级 MCP 服务、项目私有配置、密钥和环境
- 不充当真实团队项目中的运行时资产仓
