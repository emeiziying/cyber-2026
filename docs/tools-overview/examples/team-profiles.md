# 常见团队画像与工具组合

## 画像 1：VS Code 重度团队，想低成本起步

**特征：**

- 不想更换 IDE
- 以日常编码补全为主
- 暂时不做复杂自动化

**推荐组合：**

- IDE 内工具作为默认入口
- 先补共享 Rules
- 后续再补 CLI Agent 处理复杂任务

## 画像 2：已有较强工程规范，想推动复杂任务自动化

**特征：**

- 团队愿意接受 CLI 工作流
- 需要跨文件修改、批量测试、Review 辅助
- 有能力维护规则和模板

**推荐组合：**

- Agent 类工具作为复杂任务入口
- 项目级 Rules + Skills + 最小 MCP 配置

## 画像 3：管理要求严格，优先考虑数据与权限边界

**特征：**

- 对代码上传、第三方连接、Token 权限要求严格
- 更关注可控性而不是炫技

**推荐组合：**

- 先采用边界更清晰的最小能力集
- 只读优先
- 高自主能力工具延后引入

## 画像 4：混合团队，需要兼顾产品、设计和研发

**特征：**

- 不只是程序员使用 AI
- 需要统一输入、验收和协作方式

**推荐组合：**

- 研发侧用工程工具
- 非研发侧用阅读材料、模板和分享资产统一方法
- 把共享规则和验收方式放在团队文档层
