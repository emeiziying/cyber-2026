# 一个最小团队流程样例

## 任务

优化后台搜索结果页的筛选交互。

## 任务入口

任务单里先写清楚：

- 目标：减少筛选冲突带来的空结果
- 范围：只改搜索结果页，不改底层搜索接口
- 约束：保持现有权限逻辑不变
- 验收：happy path、生效失败路径、回归说明

## 执行

执行者让 AI：

1. 梳理相关文件
2. 提出最小改动方案
3. 实现并补测试
4. 用统一格式总结风险

## Review

Reviewer 重点看：

- 是否误改了权限逻辑
- 是否覆盖了筛选冲突场景
- 是否说明了兼容性风险

## 复盘

如果发现 AI 连续两次误解筛选边界，就把这条约束补进项目级 Rules。
