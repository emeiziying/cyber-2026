# 附录：案例与延伸阅读

> **适合人群：** 工程师、产品、设计、Tech Lead、少量管理者
> **学习目标：** 通过跨角色阅读材料、案例模板和复盘方法，把“会看文档”转成“会做任务”和“会沉淀经验”

---

## 为什么保留在附录？

大多数团队引入 AI 时，真正的断层不在概念理解，而在这里：

- 会看路线图，但不会把它用到真实任务里
- 会复制模板，但不知道什么时候该用哪一个
- 会让 AI 写代码，但不会做完整复盘

案例库的作用，就是把一条任务链路从输入、执行、验证到复盘走完整。

本附录承担两类补位工作：一类是跨角色阅读材料，帮助没有参加分享的人快速建立统一认知；另一类是案例模板和复盘入口，帮助团队把方法真正落到任务执行上。

## 案例与复盘模板

模板入口：

- <a href="/downloads/appendix-case-studies/templates/case-study-template.md" download><code>case-study-template.md</code></a> — 统一的案例记录和复盘结构

已有案例：

- [`fix-search-bug-case.md`](./examples/fix-search-bug-case) — 修复中文搜索失败 bug 的完整闭环案例（从定位到沉淀）

团队可以按同一模板继续补充案例，推荐优先覆盖三类场景：**修真实 bug**、**为现有模块补测试**、**做一次 PR Review**。每个案例统一按”背景 → 输入 → 执行 → 输出 → 验证 → 复盘”六步结构编写，方便横向对比和经验沉淀。

## 如何配合主线使用

如果你需要一个可直接演练的最小项目，可以配合：

- [Minimal Agent Demo](../examples/minimal-agent-demo/)

建议做法：

1. 先在 `minimal-agent-demo` 中选一个小任务
2. 按模板记录完整过程
3. 把复盘中稳定出现的问题反写进 `CLAUDE.md` 或项目级 Skills

- [ ] 演练后把经验沉淀回规则或 Skill
