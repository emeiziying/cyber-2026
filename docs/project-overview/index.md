# 内容地图

这一页用来帮助你快速判断从哪里开始读、哪些内容属于主线、哪些页面是辅助资料。正文围绕学习路径组织，模板、PPT 和示例配置则作为补充材料单独提供。

## 内容分层

| 层级 | 内容 | 入口 |
|------|------|------|
| 开始阅读 | 阅读路径、边界说明、推荐入口 | [开始阅读](/intro/) |
| 学习主线 | 从范式转变到治理与团队化的完整方法链路 | 侧边栏 `学习主线` |
| 案例与演练 | 跨角色阅读材料、案例模板、最小演练项目 | [附录：案例与延伸阅读](/appendix-case-studies/)、[Minimal Agent Demo](/examples/minimal-agent-demo/) |
| 资料与分享 | 分享大纲、PPT 成品、生成脚本和下载资源 | [分享与培训](/presentation-outlines/)、[下载资源](/downloads/) |

## 主线怎么读

推荐的学习顺序是：

1. 先读 [范式转变](/paradigm-shift/)、[Vibe Coding](/vibe-coding/) 和 [工具全景](/tools-overview/)，建立判断框架
2. 再读 [Rules](/rules/)、[MCP](/mcp/)、[Skills & Hooks](/skills-hooks/) 和 [Agent 开发](/agent-development/)，理解能力构建路径
3. 最后读 [生产落地与治理](/production-governance/) 和 [团队工作流与质量控制](/team-workflow/)，把个人做法升级成团队机制

## 辅助资料怎么用

- 想做跨角色对齐或部门培训：看 [附录：案例与延伸阅读](/appendix-case-studies/) 和 [分享与培训](/presentation-outlines/)
- 想找一个可直接演练的小项目：看 [Minimal Agent Demo](/examples/minimal-agent-demo/)
- 想拿模板、配置样例或 PPT 成品：去 [下载资源](/downloads/)

## 推荐入口

- 想理解这套方法为什么成立：从 [范式转变](/paradigm-shift/) 开始
