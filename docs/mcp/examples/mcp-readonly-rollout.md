# MCP 从只读到受控扩权的最小落地步骤

## 第一步：只读接入

先开放这类能力：

- 读取 issue / PR
- 读取指定目录文件
- 读取文档和 schema

目标是先验证上下文传递和调用链路，而不是立刻做写操作。

## 第二步：补确认点

在准备开放写能力前，先写清楚：

- 哪类操作需要人工确认
- 哪类仓库 / 目录允许被修改
- 哪些 Token 只允许最小权限

## 第三步：有限写入

如果团队确实需要写能力，建议从低风险写入开始，比如：

- 创建草稿评论
- 更新非核心文档
- 在测试环境写入临时数据

## 第四步：保留回滚与审计

一旦 MCP 会写外部系统，就必须考虑：

- 如何追溯谁触发了什么操作
- 出错后怎么回滚
- 是否需要审批或日志记录
