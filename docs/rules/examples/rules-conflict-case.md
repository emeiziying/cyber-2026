# Rules 冲突与演进示例

## 起点：团队反复纠正同一类问题

一个 React 项目里，AI 经常：

- 新建重复组件
- 用局部 state 代替全局状态
- 忽略测试文件

最初团队只是口头提醒，结果每个新会话还会重复发生。

## 第一版 Rules

团队先补了 4 条最小规则：

- 新组件前先查 `components/`
- 状态管理统一使用 Zustand
- 新增组件时必须同时补 `.test.tsx`
- 不允许使用 class 组件

## 第二个问题：安全模块不适用同一规则

后来团队发现 `src/security/` 下有更严格的约束：

- 不允许 AI 直接修改某些核心判断
- 只能先做分析和测试建议

于是他们在 `src/security/CLAUDE.md` 里补了子目录规则。

## 收获

项目级 Rules 解决的是共性，子目录 Rules 解决的是局部高风险约束。两者都需要，但职责不同。
