# Skill 与 Hook 的选择示例

## 场景 1：统一 bug 修复输出格式

**更适合：** Skill  
原因：这是用户主动发起的一类复杂流程，还需要固定输出格式。

## 场景 2：禁止在 main 分支直接写文件

**更适合：** Hook  
原因：这类约束应该自动触发，而不是靠用户记得先运行一个命令。

## 场景 3：会话开始时自动读取项目环境信息

**更适合：** Hook  
原因：这是稳定、重复、无需人工决策的初始化动作。

## 场景 4：生成日报

**更适合：** Skill  
原因：它是一个有固定输入和固定输出格式的主动任务。

## 一个快速判断

如果你在说“每次我都得手动提醒 AI 做这件事”，先问：

- 这是用户主动发起的流程吗？如果是，优先 Skill
- 这是某个事件发生时都该自动执行的吗？如果是，优先 Hook
