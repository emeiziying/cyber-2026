# 一个完整的 Vibe Coding 对话样例

## 起点：模糊需求

> 帮我加一个搜索功能。

这个说法太短，AI 很难知道范围、边界和验证方式。

## 改写后的第一轮输入

> 我有一个 Node.js + Fastify + SQLite 的笔记服务。  
> 请帮我加一个搜索接口，支持标题和内容模糊搜索。  
> 不要改已有 CRUD 行为。  
> 先告诉我你准备改哪些文件、为什么这么改。

## 第二轮追问

> 搜索结果要保持现有返回结构；如果没有结果，返回空数组，不要报错。  
> 再补充一下你准备怎么验证。

## 第三轮收敛

> 现在先实现。实现后请：
> 1. 列出你改了哪些文件  
> 2. 说明可能遗漏的边界  
> 3. 给我一组最小测试建议

## 什么时候该停下来

如果 AI 在两轮之后仍然没有说清：

- 会改哪些文件
- 为什么这样改
- 如何验证没有破坏现有行为

那就不要继续“再试一轮”。应该先回头把目标和边界讲清楚，再继续。
